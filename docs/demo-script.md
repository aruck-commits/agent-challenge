# Orion — Demo Script (50 seconds)

## Setup Before Recording

1. Start Orion with `npm run dev`
2. Open `http://localhost:3000` in a browser at 1280×800
3. Keep devtools closed and use a clean browser state
4. Keep alerts active so the feed is populated

---

## Script (timestamp → action → voiceover)

| Time | Action | Voiceover |
|---|---|---|
| 0–4s | Show dashboard loading, score ring animating | *"This is Orion — a personal Solana DeFi Risk Officer running on Nosana."* |
| 4–10s | Point to hero panel: portfolio value, score, active alerts, positions | *"I connect my wallet and immediately get a portfolio health score, alert summary, and position count."* |
| 10–18s | Click an alert card to expand it | *"Orion explains why the alert fired, what evidence it used, and what action it suggests."* |
| 18–24s | Show the positions table | *"The positions table ranks every holding by risk with reason labels."* |
| 24–30s | Click **Open Chat** in the navbar | *"There is also a dedicated chat page for live portfolio questions."* |
| 30–38s | Ask: "What is my biggest current risk?" and show the reply | *"The assistant answers with live context from the same runtime."* |
| 38–45s | Return to the dashboard and point to Insights | *"The insights panel highlights correlated exposure and rebalance suggestions."* |
| 45–50s | Zoom out to the full app | *"Orion is not just a dashboard — it's a continuous risk agent running in one container."* |

---

## Key Talking Points (if asked live)

- **Why Nosana?** Decentralised compute — the app runs 24/7 in one container.
- **Why ElizaOS?** It provides the agent framework, Solana integrations, and runtime pattern Orion uses.
- **Is the risk model AI?** The scoring layer is deterministic and explainable. AI only adds natural-language responses.
- **What's next?** Historical charts, multi-wallet aggregation, whale watch, and risk profile presets.
